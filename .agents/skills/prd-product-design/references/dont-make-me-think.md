# Don't Make Me Think — Synthesized Usability Principles

These principles are distilled from Steve Krug's *Don't Make Me Think* and guide every design decision produced by this skill.

## 1. Don't Make Me Think

A page should be self-evident, self-explanatory, and intuitive. If a user has to stop and think about:
- Where am I?
- Where do I start?
- Where is [thing]?
- Is that clickable?
- What is this for?

...the design has failed. Every element should communicate its purpose at a glance.

## 2. Conventions Are Friends

Users arrive with expectations built from other products. Follow established patterns for:
- Navigation placement and labels
- Search icon and position
- Logo linking to home
- Buttons that look like buttons
- Form field behavior and validation

Only break conventions when the new pattern is significantly better and worth the learning cost.

## 3. Effective Visual Hierarchy

Guide the eye through size, weight, color, spacing, and contrast. Every page should have:
- One clear primary action or message
- Logical grouping of related elements
- Scan-friendly structure (headings, bullets, whitespace)

## 4. Break Pages Into Clearly Defined Areas

Users should instantly understand:
- Header / navigation
- Main content
- Sidebar or secondary actions
- Footer

Clear area boundaries reduce cognitive load and help users ignore what isn't relevant.

## 5. Make Obvious What's Clickable

Links and buttons must look clickable. Use:
- Underlined or colored text for links
- Button-like shapes with clear padding
- Hover and focus states
- Consistent interactive styling

Avoid mystery meat navigation.

## 6. Minimize Noise

Every extra element competes for attention. Reduce:
- Excessive decoration
- Too many calls-to-action
- Overuse of bold, caps, or bright colors
- Unnecessary animations

If everything is loud, nothing is heard.

## 7. Format for Scanning

Users scan, they don't read. Support scanning with:
- Descriptive headings
- Short paragraphs
- Bulleted lists
- Highlighted keywords
- Clear section dividers

## 8. Get Rid of Half the Words, Then Half of What's Left

Most copy can be cut. Instructions, descriptions, and labels should be:
- Short
- Specific
- Actionable

If a word doesn't clarify or motivate, remove it.

## 9. Mindless Clicks Are Fine

It doesn't matter how many clicks a task takes as long as each click is:
- Unambiguous
- Easy to understand
- Clearly leading toward the goal

Avoid forcing hard decisions on a single screen.

## 10. The Trunk Test

At any point, a user should be able to answer within seconds:
- What site is this? (Site ID / logo)
- What page am I on? (Page name)
- What are the main sections? (Primary navigation)
- What are my options here? (Local navigation / actions)
- How do I search?

## 11. Forgiveness and Error Prevention

Design so mistakes are hard to make and easy to recover from:
- Confirm destructive actions
- Provide clear undo paths
- Validate input inline
- Use sensible defaults
- Never make a user re-enter data

## 12. Mobile: Simplify, Don't Shrink

Mobile design is not desktop squeezed smaller. It requires:
- Prioritized content
- Larger touch targets
- Simplified navigation
- Context-appropriate actions
- Reduced input friction

## 13. Test With Real Users

Usability testing beats opinion. Watch real people use the design, note where they hesitate, and fix those points first. Small tests with a few users early are more valuable than big tests late.

## Quick Design Checklist

Before finalizing any design, verify:
- [ ] Purpose of every element is immediately clear
- [ ] Primary action is the most visually prominent
- [ ] Navigation answers the trunk test
- [ ] Clickable items look clickable
- [ ] Copy is half as long as the first draft
- [ ] Page supports scanning
- [ ] Error states and empty states are considered
- [ ] Mobile experience is simplified, not shrunk
